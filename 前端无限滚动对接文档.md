# 文章列表无限滚动对接文档

## 接口说明

### GET /article

支持两种分页模式：**偏移分页**（首次加载）和 **游标分页**（无限滚动）。

### 请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pageNum | Integer | 否 | 1 | 偏移分页页码（首次加载使用） |
| pageSize | Integer | 否 | 5 | 每页条数 |
| lastId | Integer | 否 | - | 游标值，最后一条文章的 id（滚动时使用） |
| categoryId | String | 否 | - | 分类筛选 |
| state | String | 否 | - | 状态筛选（已发布/草稿） |

### 响应格式

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "total": 50,
    "items": [
      {
        "id": 1,
        "title": "文章标题",
        "content": "文章内容",
        "coverImg": "https://...",
        "state": "已发布",
        "categoryId": 1,
        "createUser": 1,
        "createTime": "2025-01-01T12:00:00",
        "updateTime": "2025-01-01T12:00:00"
      }
    ],
    "hasMore": true
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| total | Long | 总条数（仅偏移分页返回，游标分页为 null） |
| items | Array | 文章列表 |
| hasMore | Boolean | 是否还有更多数据（仅游标分页返回，偏移分页为 null） |

## 前端实现步骤

### 1. 定义状态

```javascript
const [articles, setArticles] = useState([])   // 文章列表
const [loading, setLoading] = useState(false)   // 加载中
const [hasMore, setHasMore] = useState(true)    // 是否还有更多
const [lastId, setLastId] = useState(null)      // 游标值
const [isFirstLoad, setIsFirstLoad] = useState(true) // 是否首次加载
```

### 2. 加载数据函数

```javascript
async function loadArticles() {
  if (loading || !hasMore) return
  setLoading(true)

  try {
    const params = { pageSize: 10 }

    if (isFirstLoad) {
      // 首次加载：使用偏移分页，拿到 total
      params.pageNum = 1
    } else {
      // 滚动加载：使用游标分页
      params.lastId = lastId
    }

    const res = await axios.get('/article', { params })

    if (isFirstLoad) {
      setArticles(res.data.data.items)
      setIsFirstLoad(false)
    } else {
      // 追加到现有列表
      setArticles(prev => [...prev, ...res.data.data.items])
    }

    // 判断是否还有更多
    if (res.data.data.hasMore !== null) {
      setHasMore(res.data.data.hasMore)
    } else {
      // 首次加载：根据 total 判断
      setHasMore(articles.length + res.data.data.items.length < res.data.data.total)
    }

    // 更新游标：取最后一条的 id
    const items = res.data.data.items
    if (items.length > 0) {
      setLastId(items[items.length - 1].id)
    }
  } finally {
    setLoading(false)
  }
}
```

### 3. 首次加载

```javascript
// 组件挂载或筛选条件变化时
useEffect(() => {
  setArticles([])
  setLastId(null)
  setHasMore(true)
  setIsFirstLoad(true)
  loadArticles()
}, [categoryId, state]) // 筛选条件变化时重置
```

### 4. 滚动监听（以 React 为例，可适配任何框架）

#### 方案 A：监听滚动事件

```javascript
function handleScroll() {
  const scrollTop = document.documentElement.scrollTop
  const clientHeight = document.documentElement.clientHeight
  const scrollHeight = document.documentElement.scrollHeight

  // 距离底部 100px 时触发加载
  if (scrollTop + clientHeight >= scrollHeight - 100) {
    loadArticles()
  }
}

useEffect(() => {
  window.addEventListener('scroll', handleScroll)
  return () => window.removeEventListener('scroll', handleScroll)
}, [loading, hasMore, lastId, isFirstLoad])
```

#### 方案 B：IntersectionObserver（推荐）

```html
<!-- 在文章列表底部放一个占位元素 -->
<div ref={sentinelRef} style="height: 1px" />
```

```javascript
useEffect(() => {
  const observer = new IntersectionObserver(
    ([entry]) => {
      if (entry.isIntersecting) {
        loadArticles()
      }
    },
    { rootMargin: '100px' }
  )

  if (sentinelRef.current) {
    observer.observe(sentinelRef.current)
  }

  return () => observer.disconnect()
}, [loading, hasMore, lastId, isFirstLoad])
```


## 完整流程时序

```
首次加载
  ──→ GET /article?pageSize=10
  ←── { total: 50, items: [8项], hasMore: null }
  ──→ 保存 lastId = items[last].id（如 45）
  ──→ hasMore = (当前已加载 < total) → true

用户滚动到底部
  ──→ GET /article?lastId=45&pageSize=10
  ←── { total: null, items: [10项], hasMore: true }
  ──→ 追加到列表，更新 lastId = 55

继续滚动
  ──→ GET /article?lastId=55&pageSize=10
  ←── { total: null, items: [3项], hasMore: false }
  ──→ 追加到列表，不再触发加载
```

## 注意事项

1. **排序一致性**：数据按 `id ASC` 排序，新创建的文章 ID 更大，只会出现在列表末尾，不会导致已加载数据跳动
2. **筛选变化时重置**：切换 `categoryId` 或 `state` 时，需要清空列表、重置游标、重新首次加载
3. **防重复请求**：在 `loading` 状态下不要重复发起请求
4. **`hasMore` 判断优先级**：
   - 游标分页响应中 `hasMore` 字段优先
   - 首次偏移分页响应中，通过 `total` 判断
